Conteúdo Completo — Guia Consolidado

1 — Linux (Básico ao Avançado)

O Linux é um sistema operacional de código aberto baseado em Unix, usado para servidores, desenvolvimento e computação em nuvem, oferecendo controle via linha de comando para navegação, arquivos, processos e rede.

Comandos básicos e explicações:

  • pwd: Mostra o diretório atual (Print Working Directory).
  • ls -la --color=auto: Lista arquivos e diretórios com detalhes e cores, incluindo ocultos.
  • find . -type f -name '*.log' -mtime -7: Busca arquivos .log modificados nos últimos 7 dias.
  • du -sh pasta/: Mostra o tamanho total de uma pasta em formato legível.
  • chmod +x script.sh: Torna um arquivo executável.
  • journalctl -u serviço -f: Visualiza logs de um serviço em tempo real.
  • dig +short exemplo.com: Consulta registros DNS de um domínio.
  • python3 -m http.server 8000: Inicia um servidor HTTP simples na porta 8000.

2 — Git (Básico ao Avançado)

O Git é um sistema de controle de versão distribuído para rastrear mudanças em código, colaborando em projetos e gerenciando branches.

Comandos básicos e explicações:

  • git config --global user.name "Seu Nome": Define o nome do usuário globalmente no Git.
  • git config --global user.email "seuemail@exemplo.com": Define o email do usuário globalmente.
  • git init: Inicializa um repositório Git em um diretório.
  • git add .: Adiciona todos os arquivos modificados ao stage para commit.
  • git commit -m "Início": Cria um commit com uma mensagem descrevendo as mudanças.
  • git push -u origin main: Envia commits para o repositório remoto na branch main e define upstream.

3 — Hospedagem

A hospedagem refere-se a serviços para publicar sites estáticos ou dinâmicos, como GitHub Pages, Netlify, Vercel, Cloudflare Pages e AWS S3 + CloudFront, comparados por facilidade e recursos.

Comandos básicos e explicações (exemplo GitHub Pages):

  • Settings → Pages → Branch main → Save: Configura o GitHub Pages para publicar a branch main.
  • Acesse: usuario.github.io/repositorio: URL gerada para acessar o site hospedado.

4 — HTTPS

O HTTPS é o protocolo seguro para web, usando TLS para criptografar dados, com certificados para autenticar sites e prevenir ataques.

Comandos básicos e explicações:

  • TLS 1.2 / 1.3 recomendado: Versões seguras do protocolo TLS para conexões.
  • Certificados gratuitos via Let's Encrypt: Serviço para obter certificados SSL/TLS gratuitos.
  • Strict-Transport-Security: Cabeçalho que força conexões HTTPS.
  • Content-Security-Policy: Define fontes permitidas para conteúdo.
  • X-Frame-Options: Previne carregamento em frames não autorizados.

5 — DNS

O DNS (Domain Name System) traduz nomes de domínio em endereços IP, usando registros para configurar domínios.

Comandos básicos e explicações:

  • A — aponta para IPv4: Registro que mapeia domínio para endereço IPv4.
  • AAAA — IPv6: Registro para endereço IPv6.
  • CNAME — alias: Cria um alias para outro domínio.
  • TXT — verificação / SPF: Usado para verificação e políticas de email.
  • NS — servidores de zona: Especifica servidores de nomes para a zona.

6 — Workflows de Deploy

Workflows de deploy automatizam a publicação de aplicações web, integrando com ferramentas como GitHub Pages e Vercel para builds e hospedagem.

Comandos básicos e explicações (exemplos):

  • GitHub Pages: Commit → Configurar Pages → Publicação automática: Processo para publicar via GitHub.
  • Vercel: Importar repositório → Build automático → URL gerada: Deploy automático no Vercel.

7 — Troubleshooting

O troubleshooting envolve diagnosticar e resolver erros em deploy, HTTPS, DNS e builds, usando logs e ferramentas.

Comandos básicos e explicações:

  • Certificado inválido → verificar DNS e data: Cheque registros DNS e validade do certificado.
  • DNS não propagou → ver TTL e dig +trace: Verifique tempo de vida e rastreie propagação.
  • Erro de build → revisar logs do Vercel / Netlify: Analise logs para identificar problemas.

8 — Cheatsheet Geral

O cheatsheet geral resume comandos essenciais de Linux, Git e rede para referência rápida.

Comandos básicos e explicações:

  • Linux: ls -la (lista arquivos), cd (muda diretório), rm -rf pasta/ (remove pasta recursivamente).
  • Git: git add . (adiciona mudanças), git commit -m "msg" (commit com mensagem), git push (envia para remoto), git pull (baixa mudanças).
  • Rede: curl -I https://site.com (verifica cabeçalhos HTTP), dig +short dominio.com (consulta DNS rápida).
